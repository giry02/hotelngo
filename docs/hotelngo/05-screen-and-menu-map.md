# HotelNGo 화면·메뉴·상태 명세

## 1. 원칙

- B2C, 파트너센터, 플랫폼 관리자를 별도 앱 셸과 인증 경계로 운영한다.
- 웹과 모바일은 같은 도메인/API를 사용하되 탐색 구조는 각 기기에 맞춘다.
- 모든 목록은 로딩, 비어 있음, 부분 실패, 권한 없음, 재시도 상태를 가진다.
- 가격/재고 화면은 정상, 매진, 공급자 미응답, 오래된 캐시를 서로 다른 문구와 시각 상태로 표시한다.
- 공개 화면에는 공급 시스템의 물리 호실 ID, 직원, Folio, 내부 메모를 표시하지 않는다.

## 2. B2C 웹 정보구조

### 2.1 전역 메뉴

| 영역 | 메뉴/기능 |
|---|---|
| 기본 메뉴 | `홈 / 여행 발견 / 호텔 / 즐길거리 / AI 여행 / 내 여행 / 마이` |
| 여행 발견 | 여행 이야기, 랜드마크, 테마, 도시 가이드, 호텔에서 시작하는 여행, 골프·휴양 |
| 즐길거리 | 골프, 맛집·카페, 마사지·스파, 투어·체험, 렌터카·픽업 |
| 공통 액션 | 로고, 통합검색, 언어/통화, 로그인, 찜, 장바구니, 알림 |
| 마이 | 내 일정, 예약/주문, 리뷰, 프로필/동행자 |
| 푸터 | 회사/서비스, 파트너 입점, 고객센터, 이용약관, 개인정보, 취소/환불, 접근성, 언어/통화 |

### 2.2 홈

홈은 원문 우선순위에 따라 다음 순서를 기본으로 하되 운영자가 기능 플래그와 노출 순서를 조정할 수 있다.

1. 대표 여행 스토리 히어로
2. 테마 여행
3. 30초 여행 영상
4. 랜드마크 탐색
5. 호텔 추천/검색
6. 골프·휴양
7. 미식·야간 여행
8. AI 여행 진입
9. 축약 호텔 검색
10. 예약 가능 상품
11. 최근 본 여행·저장 콘텐츠

히어로는 영상/이미지, 제목, 내레이션, 도시·지역, 기간 또는 하루 흐름, 스토리 보기, 여행 진입, 내 여행 저장 CTA를 제공한다. 탐색 몰입을 위해 예약 CTA는 스토리 전반이 아니라 후반에 배치한다.

### 2.3 공통 검색 결과

- 검색어/목적지, 날짜·시간, 인원, 카테고리 탭
- 지도/목록 전환, 정렬, 필터, 결과 개수
- 카드: 대표 이미지, 이름, 위치, 평점/리뷰, 핵심 태그, 최저가 또는 정보성 상태, 최신 확인시각
- 모바일에서는 필터를 바텀시트로, 지도와 목록을 전환형으로 제공한다.

## 3. 스토리와 랜드마크

### 3.1 스토리 화면

| 화면 | 구성 |
|---|---|
| 스토리 목록 | 테마·지역·기간·동행·예산 필터, 추천/인기/신규 정렬 |
| 스토리 상세 | 표지, 제목/요약, 작성자/출처, 지역·기간·예산, 블록 본문, 연결 장소/상품, 지도, 저장/공유/일정담기 |
| 내 스토리 | 초안/검토중/발행/반려 상태, 복제/수정/제출 |
| 편집기 | 제목·요약·메타와 블록 추가/이동/삭제, 미리보기, 출처 표시 |

지원 블록은 텍스트, 이미지, 이미지 갤러리, 인용, 팁/주의, 장소, 랜드마크, 호텔/상품, 지도, 일정, 이동, 비용 요약, CTA로 한다. AI 생성 블록은 생성 이력과 근거를 편집 화면에서 확인할 수 있어야 한다.

### 3.2 랜드마크/장소 화면

- 목록/지도 검색: 지역, 카테고리, 무료/유료, 운영시간, 실내/야외, 가족/접근성 태그
- 상세: 이름/설명/미디어/좌표/주소/시간대/운영시간/입장료/휴무/연락처/공식 링크/접근성/주의사항/최신 확인시각
- 주변 호텔·음식점·체험·교통, 관련 스토리, 일정 추가
- 폐업/일시중단/검증필요 상태는 숨기거나 명시적으로 표시한다.

| 객실·요금 | 객실유형, 기준/최대인원, 침대, 면적, 전망, 편의, 식사, 취소조건, 세금 포함 여부, 총액, 잔여수량, 확인시각 |

| 업종 | 상세/예약 핵심 필드 |

| 렌터카 | 픽업/반납, 차급, 승차/수하물, 연료, 주행거리, 보험, 보증금, 운전자 조건 |

## 7. 모바일 내비게이션

하단 탭은 `홈 / 검색 / AI 여행 / 내 여행 / 마이`를 기본으로 한다. 검색 탭은 호텔, 랜드마크, 골프, 맛집, 마사지, 차량, 투어 탭을 제공한다. 장바구니와 알림은 헤더 액션으로 제공하고, 예약은 내 여행 또는 마이에서 접근한다. 웹의 모든 거래 상태와 정책 정보가 모바일에서도 누락되지 않아야 한다.

## 8. 파트너센터

### 8.1 공통 메뉴

| 그룹 | 메뉴 |

| 시작 | 대시보드, 온보딩/심사, 공지/할 일 |
| 사업장 | 업체/지점, 공개정보, 영업정보, 위치, 미디어, 소유권 클레임 |
| 판매 | 상품/옵션, 가격/재고/캘린더, 판매제한, 정책, 프로모션 |
| 운영 | 예약, 변경/취소, 고객 문의, 리뷰/답변, 알림 |
| 재무 | 매출, 정산, 지급계좌, 세금/문서 |
| 조직 | 구성원, 역할, 초대, 활동 이력 |
| 설정 | 연동, 알림, 언어, API 자격증명(허용 파트너만) |

### 8.2 주요 화면

- 온보딩 단계: 계정 → 사업자/업체 → 지점 → 업종별 필수정보 → 정산 → 문서 → 제출 → 심사결과
- 대시보드: 오늘/다가오는 예약, 미처리 요청, 판매 중지/재고 경고, 매출/전환, 최신성 갱신 요청
- 상품 편집: 공통 필드와 업종별 필드, 저장/미리보기/심사제출
- 가격·재고: 달력/목록, 일괄 변경, 제한조건, 변경 영향/충돌 검증
- 예약 상세: 고객/상품/일정/금액/정책/이력, 허용된 상태 명령
- 변경 요청: 현재 공개값과 요청값의 비교, 근거, 심사 상태

## 10. 공통 UI 상태와 품질

- 로딩은 스켈레톤/진행 상태를 사용하고 중복 제출을 막는다.
- 오류는 사용자가 할 수 있는 다음 행동을 포함한다.
- 오래된 공급 데이터는 `마지막 확인 시각`과 함께 경고하고, 판매 기준을 넘으면 예약을 차단한다.
- 접근성: 키보드 탐색, 포커스 표시, 명시적 레이블, 대비, 오류 연결, 대체텍스트, 동작 감소 설정을 적용한다.
- 국제화: 문자열 하드코딩 금지, 날짜/시간대/숫자/통화/주소 형식 현지화, 원문과 번역 상태 구분.
- SEO: 공개 상세의 고유 URL, canonical, 구조화 데이터, 언어 대체 링크, 서버 렌더링/사전 렌더링을 Release 2 이후 적용한다.
