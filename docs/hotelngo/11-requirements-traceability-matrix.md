# HotelNGo 요구사항 추적표

## 1. 목적과 사용법

이 표는 첨부 원문, 마스터 요구사항, 설계 문서, 구현 릴리스, 검증 증거를 연결한다. 요구사항의 정본은 `00-requirements-specification.md`이며, 구현 티켓·브랜치·커밋·테스트 이름에는 최소 하나의 요구사항 ID를 포함한다.

상태 값은 다음으로 고정한다.

- `PROPOSED`: 명세됐으나 구현 릴리스 미착수
- `READY`: 의존 결정과 인수 기준이 준비됨
- `IN_PROGRESS`: 구현/검증 진행 중
- `VERIFIED`: 구현과 인수 증거가 승인됨
- `DEFERRED`: 명시된 후속 릴리스로 연기
- `BLOCKED`: 외부 의존성과 해제 조건이 기록됨

문서 작성만으로 요구사항을 `VERIFIED`로 바꾸지 않는다.

## 4. 릴리스별 요구사항 기준선

### Gate 0

`DEL-001~003`, `PMS-001`, `SYS-001~005`를 `READY`로 만들고 나머지 요구사항의 우선 릴리스와 의존성을 승인한다.

### Release 1

- 구현 기준: `SUP-001~005`, `DOM-001~005`, `HTL-001~006`, `API-001`, `PMS-001~004`, 관련 `NFR/ACC`
- 명시적 제외: 예약 쓰기, 결제, 파트너/관리자 실인증, AI
- 인수 증거: Mock 계약, 검색/상세/Offer E2E, 격리/비노출, PMS 변경 없음

### Release 2

- 구현 기준: `IAM-*`, `PTN-*`, `CUR-*`, `CLM-*`, `DQ-*`, `ADM-001~003`, `API-002~003`, 관련 `VRT/DAT/NFR`
- 인수 증거: API/DB/auth, 파트너 심사, 선등록/중복, 클레임, 최신성, 감사로그

### Release 3

- 구현 기준: `HTL-007~008`, `PMS-005~008`, 호텔 관련 `COM-*`, `API-004`, 관련 `NFR/ACC`
- 인수 증거: Shadow 게이트, hold/예약/변경/취소/대사, PMS 회귀

### Release 4

- 구현 기준: `VIS-*`, `B2C-*`, `STO-*`, `PLC-*`, `ITN-*`, 정보성 `VRT-*`, 관련 `ACC`
- 인수 증거: 탐색형/목적지형 홈, 스토리/장소/일정 E2E, 콘텐츠 공개/회수

### Release 5

- 구현 기준: 거래형 `VRT-*`, `COM-*`, `SUP-006~009`, `API-004`, 관련 `NFR/ACC`
- 인수 증거: 업종 계약, 통합 장바구니, 부분확정/환불/정산, 장애 복구

### Release 6~7

- 구현 기준: `AIR-*`, `API-005`, `ADM-004`, `STO-*`, `ITN-*`, 관련 `NFR/ACC`
- 인수 증거: RAG 삭제/권한/근거, 일정 제약, 실시간 재검증, AI 거래 금지, 발행 승인

## 5. 구현 티켓과 PR 규칙

각 구현 티켓은 다음 형식을 사용한다.

```md
제목: [HTL-003][PMS-003] 객실유형 Offer 조회 구현

- 범위: 포함/제외
- 선행조건/가정:
- 데이터/API 변경:
- 화면/권한 영향:
- 실패·빈 상태·롤백:
- 인수 기준:
- 테스트: 단위/계약/통합/E2E
- 문서 변경:
```

PR 체크리스트:

- 요구사항 ID와 릴리스가 적혀 있는가
- API/스키마/상태 전이/권한 변화가 문서와 테스트에 반영됐는가
- 다른 파트너/테넌트/사용자 범위를 검사했는가
- Mock이나 임시 가정이 명시됐는가
- 가격/재고/예약/결제의 허위 성공 경로가 없는가
- 관측·마이그레이션·기능 플래그·복구 영향이 기록됐는가

## 6. 자동 누락 검사

PowerShell에서 다음 읽기 전용 검사를 실행한다.

```powershell
$master = rg -o "[A-Z0-9]{2,3}-[0-9]{3}" docs/hotelngo/00-requirements-specification.md | Sort-Object -Unique
$all = rg -o "[A-Z0-9]{2,3}-[0-9]{3}" docs/hotelngo | Sort-Object -Unique
Compare-Object $master $all
$master.Count
```

범위 연속성은 `B2C-006~009`의 의도된 공백을 제외하고 접두사별 번호를 검사한다. 문서 링크는 README의 모든 대상 파일 존재 여부를 확인한다. 구현이 시작되면 별도 요구사항 상태 레지스트리 또는 이슈 트래커에서 각 개별 ID의 상태와 증거 URL을 관리한다.
