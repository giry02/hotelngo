## 4. 구현 상태 해석

- 화면 전환과 입력 검증은 피드백을 위한 클라이언트 프로토타입이다.
- 결제 버튼은 PG 승인, 카드 토큰화, 금전 거래를 실행하지 않는다.
- B2C와 파트너 로그인·회원가입·비밀번호 재설정/변경은 서로 다른 JSON Fixture와 브라우저 Session/Local Storage로 동작한다. 서버 토큰, OAuth, 이메일 발송, MFA는 생성하지 않는다.
- 파트너 가입·신청·심사·선등록·Claim은 JSON/Local Storage 상태 전이 검증용이며, 실제 사업자 문서는 파일 내용 대신 참조 이름만 기록한다.
- 관리자 로그인은 B2C와 다른 JSON Fixture와 Session Storage 키를 사용하며, PMS 직원 계정을 수용하지 않는다.
- 여행자 기본값은 마스킹된 여권번호만 보관하는 Mock이다. 실서비스 암호화 저장소를 대신하지 않는다.
- 예약 완료 화면은 PMS 쓰기 결과가 아니라 `PENDING_SUPPLIER` 상태 표현 예시다.
- Hotel_PMS 파일이나 `localStorage`는 런타임 공급 데이터로 읽지 않는다.

## 5. 실구현 전 남은 게이트

- B2C 서버 인증·이메일 발송과 파트너·관리자·PMS 채널 SSO/MFA·권한 API
- 파트너 사업자 문서 저장·검수, Provider/Place/Product/Offer/Inventory DB, tenant/partner/provider 서버 격리
- 결제대행 어댑터, 결제·환불 상태 머신과 웹훅
- 재고 재검증, hold, 멱등 예약 명령과 PMS 채널 API
- 주문·예약 대사, 재시도 작업, 감사 로그와 운영 복구
- 지도·알림·검색·AI/RAG 외부 어댑터
