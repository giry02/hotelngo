## 3. 액티비티 파트너

### 상품 구성

`partner-products.html`에서 골프 상품을 다음 판매 단위로 확인했다.

- Ocean Course 18홀 · 178,000원/1인
- Mountain Course 18홀 · 최저 196,000원/1인
- Ocean Twilight 9홀 · 98,000원/1인

상품 등록 화면에서 상품명, 상품 유형, 연결 자원, 설명, 가격 방식, 가격 단위, 포함 사항과 취소 정책을 입력한 뒤 초안 저장과 목록 복귀를 확인했다.

### 자원·슬롯·결합 혜택

- 골프: 코스, 18/9홀, Par, 난이도, 티타임당 수용 인원, 캐디·카트·장비 대여
- 차량: 차종, 승객·수하물, 기사·보험, 이용 시간
- 음식점: 메뉴·코스, 좌석·룸, 예약 시간
- 마사지·스파: 프로그램, 소요시간, 트리트먼트 룸, 픽업
- 투어: 코스, 회차, 언어, 집결·픽업, 수용 인원

`partner-resources.html`, `partner-schedules.html`, `partner-bundles.html`에서 자원 등록, 슬롯 생성, 호텔 결합 혜택과 비용 분담 입력 구조를 확인했다.

### 고객 선택과 예약·정산

1. B2C 골프 상세에서 코스·홀·티타임·옵션·호텔 결합 할인을 선택한다.
2. 선택한 항목이 여행 카트에 추가되는 것을 확인한다.
3. 파트너 예약 목록에서 고객, 상품, 자원, 슬롯, 옵션, 결합 할인과 결제액을 확인한다.
4. 예약 상세에서 슬롯 차감과 상태 이력, 할인 분담과 지급 예정액을 확인한다.
5. 정산 화면에서 예약별 고객 결제, 파트너 할인 분담, 수수료, 환불·조정과 지급 예정액을 확인한다.

실제 슬롯 차감, PG 승인, 공급 예약, 환불과 은행 지급은 연결하지 않았다.

## 4. 회원 일정과 커뮤니티

### 검증한 흐름

1. 공개 회원 일정 목록에서 지역·테마를 탐색한다.
2. 일정 상세에서 작성자, 날짜별 코스와 항목별 예약 방식을 본다.
3. 비로그인 상태에서 “내 여행으로 복사”를 누르면 B2C 로그인으로 이동한다.
4. 로그인 후 원본과 다른 ID의 독립 초안을 생성한다.
5. 일정 편집기가 복사 출처와 날짜별 코스를 불러온다.
6. 공개 범위와 개인정보 제외 규칙을 선택해 공개 심사를 요청한다.
7. 예약 준비 화면에서 각 항목을 즉시예약, 요청예약, 정보형, 확인 필요로 구분한다.

플랫폼 관리자는 `admin-community.html`에서 회원 일정의 작성자가 아니라 공개 승인·수정 요청·신고 검토자 역할만 가진다.

## 5. 수정한 공통 동작

- 전용 폼의 제출 버튼에 공통 백오피스 작업 팝업이 중복으로 붙지 않도록 분리했다.
- 여행 일정 발행 화면의 여행지 입력을 호텔 통합검색 입력으로 오인하던 선택자 범위를 제한했다.
- JSON fixture 요청에 `no-store`를 적용해 변경된 데모 계정과 데이터가 이전 캐시로 보이지 않게 했다.
- 액티비티 파트너 메뉴에서 호텔 운영을 제거하고 별도 호텔 콘텐츠센터로 안내한다.

## 6. 실제 서비스 전환에 남은 외부 의존

다음은 화면이 존재해도 실서비스 구현 완료가 아니다.

- 서버 인증, 이메일 확인, MFA, RBAC, 테넌트 격리와 감사 로그
- 관계형 Provider/Place/Product/Resource/Schedule/Offer/Inventory/Booking DB
- 이미지 업로드, 권리 확인, 리사이즈, CDN과 삭제 보존 정책
- Hotel_PMS `/api/v1/channel/*` 조회, hold, 예약 생성·변경·취소·대사
- 골프·차량·음식점·스파·투어 공급자 또는 채널 API
- PG 결제·부분환불·Webhook과 정산 원장·세금·은행 지급
- 지도·경로·영업시간·알림·검색 인덱스 어댑터
- LLM/RAG, 실시간 공급 도구, 일정 충돌 검증과 AI 평가 저장소
- 회원 공개 콘텐츠의 실제 심사·신고·저작권·개인정보 처리 백엔드

화면 Mock은 위 외부 기능의 계약과 사용자 피드백을 검토하는 기준선이며, 실제 거래 완료 증거로 사용하지 않는다.
